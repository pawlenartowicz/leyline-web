# Editing in the browser

> [!info]
> **This is a future expansion.** The capability described below is in development, currently aimed at roughly 2–5 months out — the least certain item on the roadmap. Today, edit via the Obsidian plugin, the `leyline` CLI, or any text editor on the synced directory.

## What it will be

Open a note in the web reader, edit it in place, hit save. The change goes through the same sync wire the desktop plugin and CLI already use, so it shares one data path with the rest of the system.

The parity goal is small fixes: typos, link tweaks, a sentence here and there, without needing to switch into Obsidian.

## Why it matters

Nothing to install for a quick edit. The web reader becomes an editor for anyone with edit access (an API key carrying the edit capability — the permission attached to your key by your vault admin) — handy for drive-by contributions, and for editing from a phone before a mobile app exists.

## The honest caveat

Web editing is uncertain because the editing surface needs to be good enough to be worth using, without trying to compete with Obsidian for heavy-lift work. It ships when the small-fix experience feels right.
